Understanding How Backup Works

The database backup is managed by navicat:

The basic steps are as follows:

  • backup
  • archival
  • restore

running the backup

the navicat software is configured to create a backup file once a week and store it in "/Users/imagetracdb/Library/Application Support/PremiumSoft CyberTech/Navicat for PostgreSQL/localhost/b5_db/public/"
it runs Thursdays at 6pm:
backups can be manually run by clicking the "Backup" menu bar icon:
and pressing the "Start" icon in the footer bar:

archiving the result

the imagetracdb server station runs a scheduled job that copies new backup files from the local navicat location to the "/Volumes/ADVERTISING/Inserts/IMAGETRAC/backup/" directory. This is the network backup archive.

restoring from backup

the available backup jobs are displayed under the "Backup" menu tab (make sure "Backups" is also selected on the sidebar at left):
if the backup you'd like to restore isn't displayed, it likely isn't in the local navicat environment. Drag a copy from the network archive (/Volumes/ADVERTISING/Inserts/IMAGETRAC/backup/)...
...to the local navicat backup location (/Users/imagetracdb/Library/Application Support/PremiumSoft CyberTech/Navicat for PostgreSQL/localhost/b5_db/public/). Then refresh the database by clicking the "Refresh" icon in the footer - or by control-clicking the b5_db icon in the sidebar, and closing/re-opening the database:
now double-click the backup you'd like to restore, then click "Start" in the footer:
